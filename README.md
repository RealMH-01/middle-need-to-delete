# 订单文件夹自动创建工具

一款专为业务员 / 跟单员设计的 Windows 桌面工具，用于一键生成标准订单目录结构、
自动复制模板文件并重命名、自动生成文件清单 Excel。支持外贸 / 内贸、单笔 / 批量、
多业务员 / 多客户 / 多模板管理。

- 开发语言：Python 3.9+ / PyQt5
- 依赖库：PyQt5、openpyxl、pyinstaller
- 适用平台：Windows（同样可在 Linux / macOS 以源码方式运行）

---

## 一、功能一览

| 模块 | 功能 |
| --- | --- |
| 启动页 | 设置"公司资料根目录"、"模板文件目录"、一键扫描导入业务员 |
| 单笔创建 | 选择业务员 / 客户 / 订单类型 / 模板 / 产品类别 → 填表 → 预览 → 创建 |
| 批量导入 | 支持 Excel 导入 / 手动添加多行一次创建 |
| 模板管理 | 新建 / 编辑 / 另存为个人或客户模板 / 删除 / 预览 |
| 历史记录 | 记录每次创建结果，支持搜索过滤、详情查看、以此新建 |
| 使用帮助 | 侧边面板形式的傻瓜式图文教程，可与操作页面并排显示，支持章节快速跳转 |
| 首次设置向导 | 全新安装时自动弹出四步引导，帮助用户完成根目录选择、订单文件夹确认、产品类别配置、业务员导入 |

创建时自动完成：
1. 按模板创建所有文件夹（已存在的会自动跳过，缺失的会补建）。
2. 把模板文件目录中的模板文件复制到目标位置，并按规则重命名。
3. 在订单号文件夹内生成 `文件清单-<订单号>.xlsx`，可手动打勾。
4. 写入操作历史。

### 订单文件夹生成位置

```
<根目录>/<订单根文件夹>/<业务员rel_path>/[<mid_layer>/]<客户名>/<订单号>/
                                                              ├── SD/
                                                              ├── 报关资料/
                                                              ├── 货代资料/
                                                              ├── ...
                                                              └── 文件清单-<订单号>.xlsx
```

- **订单根文件夹**：默认 `1订单`，可在首页「⚙ 高级设置」中改成你们公司的实际名字（如 `Orders`、`订单管理`）。
- **业务员 rel_path**：可以是 `张三` 这种直接路径，也可以是 `华南分公司/赵六` 这种分公司下的路径。
- **mid_layer**：若业务员文件夹下含有同时包含所有中间层关键词（默认 `进行`、`订单`）的子文件夹（如 `进行中订单`、`1.进行订单`），则作为中间层保留；否则为空。关键词可在高级设置中修改。
- 具体结构由程序首次启动时自动扫描生成（见下方"扫描导入业务员"）。

## 二、使用方法（面向"从没装过 Python"的小白）

### 第一步：安装 Python

1. 打开浏览器访问 <https://www.python.org/downloads/>
2. 点击 **Download Python 3.x.x**（黄色大按钮）。
3. 下载完成后双击安装包。
4. **⚠ 最关键：在安装向导的第一页，一定要勾选 **`Add Python to PATH`** 这个复选框**，然后点击 **Install Now**。
5. 等安装完成，点击 Close。

> 如果忘了勾选 `Add Python to PATH`，请把 Python 卸载后重新安装，务必勾上。

### 第二步：下载本项目

方式一（推荐小白）：
1. 打开项目的 GitHub 页面。
2. 点击绿色的 **Code** 按钮 → **Download ZIP**。
3. 下载后解压到一个你熟悉的位置，例如桌面。

方式二：
```bash
git clone <仓库地址>
```

### 第三步：打包为 exe

在项目根目录下打开 cmd 或 PowerShell，执行以下命令：

```bash
pip install pyinstaller
pyinstaller --noconfirm --onefile --windowed --name "订单文件夹工具" main.py
```

打包完成后，可执行文件位于 `dist/订单文件夹工具.exe`。

### 面试/演示环境搭建

运行以下命令一键生成模拟数据：

```bash
python generate_demo_data.py
```

脚本会在项目根目录下创建 `demo_company/`，包含：

- 订单根文件夹 `1订单/` 下的 5 位业务员（其中 2 位位于「华南分公司」下）、多个客户与订单；
- `模板文件/` 目录下的空模板文件（含通用 / 外贸通用 / 华北工厂 / 华南工厂）；
- `.order_tool/` 目录下的 `config.json`、`salespersons.json`、`history.json` 及标准模板；
- 同时会把 `~/.order_tool_bootstrap.json` 指向这个演示目录。

生成完成后，直接运行 `python main.py` 即可体验完整功能，无需手动配置。

也可以指定生成目录：

```bash
python generate_demo_data.py /path/to/your_demo_dir
```

### 第四步：运行

1. 进入项目下的 `dist` 文件夹。
2. 双击 **`订单文件夹工具.exe`** 即可运行。
3. 可以把这个 exe 复制到任何 Windows 电脑上使用（不需要再装 Python）。

### 第五步：首次使用

> **如果是全新安装**，程序第一次打开时会自动弹出设置向导，引导你完成下面这些配置。
> 你只需要按向导提示操作即可，不需要手动执行以下步骤。
> 以下说明是给需要手动修改配置的情况准备的。

1. 打开程序后，先在首页设置：
   - **公司资料根目录**：程序会在其下的订单根文件夹（默认 `1订单/`）里创建订单目录结构。
   - **模板文件目录**：你预先准备好的模板文件所在目录（可选，不设置则只创建空文件夹）。
2. 点击对应的 **保存** 按钮。
3. 点击 **🧭 扫描导入业务员** 按钮，程序会自动读取订单根文件夹下的文件夹，弹出勾选窗口：
   - 勾选属于业务员的文件夹（如「张三」「李四」）。
   - 对于分公司（如「华南分公司」），展开后勾选其中的业务员（如「赵六」）。
   - 不勾选非业务员目录（如「技术认证-滨海仓库」）。
   - 程序会自动识别业务员下的「进行中订单」「1.进行订单」等中间层，把其中的子文件夹作为客户导入。
4. 如果你们公司的订单文件夹名、产品类别、工厂名称跟默认的不一样，可以点 **⚙ 高级设置** 按钮，
   一次性自定义这些参数，不用改代码（详见"六、高级设置"）。
5. 然后点「📝 单笔创建」或「📦 批量导入」开始使用。
6. 需要帮助时点击 **❓ 使用帮助**，有完整的图文教程。

## 三、模板文件准备（可选，但强烈推荐）

默认按下列目录结构准备 10 个模板文件（扩展名需与高级设置中的「产地文件扩展名映射」一致）：

```
<你选择的"模板文件目录">/
├── 通用/
│   └── CG.xlsx                      （采购合同通用模板）
│
├── 外贸通用/
│   ├── CI.xlsx                      （商业发票）
│   ├── PL.xls                       （装箱单）
│   └── 托书.doc                     （订舱托书）
│
├── 华北工厂/                        （默认产品类别=环氧树脂 时使用）
│   ├── 华北工厂外贸生产.doc
│   ├── 华北工厂外贸发货.docx
│   ├── 华北工厂内贸生产.xlsx
│   └── 华北工厂内贸发货.xlsx
│
└── 华南工厂/                        （默认产品类别=其他产品 时使用）
    ├── 华南工厂外贸生产.xlsx
    └── 华南工厂外贸发货.xlsx
```

> 子目录名称和文件扩展名都由 `config.json` 控制（见"六、高级设置"）。
> 如果你们公司的工厂名称/产品类别不一样，改高级设置里的映射即可，**不用改代码**。

创建订单时，程序会根据「订单类型 + 产品类别」自动挑选对应模板：

| 订单类型 | 产品类别 | 会复制的模板（默认映射下） |
| --- | --- | --- |
| 外贸 | 环氧树脂 | 通用/CG.xlsx + 外贸通用/CI.xlsx + 外贸通用/PL.xls + 外贸通用/托书.doc + 华北工厂/华北工厂外贸生产.doc + 华北工厂/华北工厂外贸发货.docx |
| 外贸 | 其他产品 | 通用/CG.xlsx + 外贸通用/CI.xlsx + 外贸通用/PL.xls + 外贸通用/托书.doc + 华南工厂/华南工厂外贸生产.xlsx + 华南工厂/华南工厂外贸发货.xlsx |
| 内贸 | 环氧树脂 | 通用/CG.xlsx + 华北工厂/华北工厂内贸生产.xlsx + 华北工厂/华北工厂内贸发货.xlsx |
| 内贸 | 其他产品 | 通用/CG.xlsx（华南工厂默认无内贸模板） |

找不到文件会自动跳过，不影响文件夹创建。

## 四、命名规则自定义

在【模板管理】 → 选模板 → 编辑 → 右侧表格中，可以自由修改每个文件的 `filename` 字段。
支持以下占位符（创建时会自动替换为真实值）：

| 占位符 | 含义 | 示例 |
| --- | --- | --- |
| `<订单号>` | 表单中填写的订单号 | `ORD-2026001` |
| `<客户名称>` | 表单中填写的客户名称 | `示例客户A` |
| `<客户PO号>` | 客户 PO 号 | `PO-12345` |
| `<产品信息>` | 产品信息 | `产品示例 200KG` |
| `<业务员>` | 当前选择的业务员姓名 | `张三` |
| `<日期>` | 创建当天日期 YYYYMMDD | `20260421` |
| `<自定义编号>` | 自定义编号（可选） | |

示例：

- `CI-<订单号>.xlsx` → `CI-ORD-2026001.xlsx`
- `销售合同-<客户名称>-<日期>.pdf` → `销售合同-示例客户A-20260421.pdf`

此外，`file_template` 字段支持 `[产地]` 标记，会根据当前订单的「产品类别」查找 `origin_map` 里对应的产地，再拼上文档类型查扩展名：

- `[产地]外贸生产` 在默认映射下，产品类别=环氧树脂 时变成 `华北工厂/华北工厂外贸生产.doc`
- `[产地]外贸发货` 在默认映射下，产品类别=其他产品 时变成 `华南工厂/华南工厂外贸发货.xlsx`

这些映射可以在首页「⚙ 高级设置」中自定义，无需改代码。

## 五、高级设置（不用改代码即可通用化）

点击首页底部的 **⚙ 高级设置** 按钮，可以一次性调整以下四项配置，满足不同公司的实际场景：

| 设置项 | config.json 字段 | 默认值 | 说明 |
| --- | --- | --- | --- |
| 订单根文件夹名 | `order_root_folder` | `"1订单"` | 订单所在的一级文件夹名称（如 `1订单`、`Orders`、`订单管理`）。 |
| 中间层关键词 | `mid_layer_keywords` | `["进行", "订单"]` | 同时包含这些关键词的文件夹会被识别为"中间层"；留空则不启用中间层识别。 |
| 产品类别与产地映射 | `origin_map` | `{"环氧树脂": "华北工厂", "其他产品": "华南工厂"}` | 影响产品类别下拉框的可选项，以及 `[产地]` 标记的解析。首次设置向导中如果选了"不需要产品类别"，此项为空，单笔创建和批量导入页面中产品类别下拉框会自动隐藏。 |
| 产地文件扩展名映射 | `origin_file_ext` | 8 项，见 `config.json` | 每种`产地/文档类型`组合对应的模板文件扩展名。 |

修改后点「保存」即生效，单笔创建、批量导入、扫描导入等页面会立即按新配置刷新。

> **升级兼容**：旧版本的 `config.json` 缺少这些字段时，程序启动后会自动补写默认值，不需要手动迁移。

## 六、数据存放位置

所有数据以 JSON 形式保存在你选择的 **根目录** 下的 `.order_tool/` 子目录中：

```
<根目录>/.order_tool/
├── config.json            程序配置（模板目录、上次选择等）
├── salespersons.json      业务员和客户列表（含 rel_path / mid_layer）
├── history.json           历史记录
└── templates/             模板文件夹
    ├── standard_export.json
    ├── standard_domestic.json
    ├── <业务员>_default_export.json      业务员个人外贸模板
    ├── <业务员>_default_domestic.json    业务员个人内贸模板
    └── <业务员>_<客户>_<type>.json       客户专属模板
```

### salespersons.json 新结构（v2）

```json
{
  "list": [
    {
      "name": "张三",
      "rel_path": "张三",
      "mid_layer": "",
      "customers": ["示例客户A", "示例客户B"]
    },
    {
      "name": "赵六",
      "rel_path": "华南分公司/赵六",
      "mid_layer": "",
      "customers": ["客户A", "客户B"]
    },
    {
      "name": "李四",
      "rel_path": "李四",
      "mid_layer": "进行中订单",
      "customers": ["客户X", "客户Y"]
    }
  ]
}
```

- `rel_path`：业务员目录相对于订单根文件夹的路径，支持「华南分公司/赵六」这种分公司写法。
- `mid_layer`：中间层文件夹名，例如「进行中订单」「1.进行订单」。留空表示没有中间层。
- 旧版（只有 `name` 和 `customers`）会在首次读取时自动补全缺失字段。

> 想在多台电脑上共用，可以把根目录放到共享盘或云盘里。

## 七、常见问题

**Q：双击 exe 没反应？**
用记事本打开 `dist/订单文件夹工具.exe.log`（如果有），或直接用命令行运行查看错误。

**Q：怎么备份数据？**
直接把 `<根目录>/.order_tool/` 整个文件夹复制到其它地方即可。

**Q：可以修改公司标准模板吗？**
可以在【模板管理】中"编辑"，但会强制你另存为"个人模板"或"客户模板"，标准模板文件本身不会被覆盖。

**Q：我们公司的订单文件夹不叫「1订单」，也没有「进行中订单」这样的中间层，怎么办？**
点首页 **⚙ 高级设置**，在里面一次性改掉订单根文件夹名、中间层关键词、产品类别与产地映射，保存即可生效，全程不用改代码。
